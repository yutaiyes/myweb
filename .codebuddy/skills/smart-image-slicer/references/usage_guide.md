# 智能图像切片工具使用指南

## 工具概述

智能图像切片工具使用计算机视觉技术自动检测并分离图像中的多个独立物体，特别适用于：

- 游戏精灵图集的自动分离
- 角色动作帧的批量提取
- UI元素的自动切片
- 手绘图稿的数字化处理

## 核心算法原理

### 1. 轮廓检测 (Contour Detection)
- 使用OpenCV的二值化处理将图像转换为黑白图
- 通过`cv2.findContours()`识别所有独立的封闭区域
- 使用`RETR_EXTERNAL`模式只检测外轮廓，避免嵌套干扰

### 2. 智能过滤
- **面积过滤**: 通过`min_area`参数过滤掉噪点和阴影
- **边界框计算**: 使用`cv2.boundingRect()`获取每个物体的最小包围矩形

### 3. 背景去除
- **局部去底**: 对每个切片单独进行背景色分析
- **容差匹配**: 使用RGB差值计算，支持渐变背景
- **透明度处理**: 将背景像素设置为完全透明

## 参数调优指南

### min_area (最小面积阈值)
- **默认值**: 500
- **作用**: 过滤小噪点和细节
- **调优建议**:
  - 图像中有很多小装饰 → 增大到 800-1500
  - 需要保留小物体 → 减小到 200-300
  - 高分辨率图像 → 按比例增大

### tolerance (背景色容差)
- **默认值**: 40
- **作用**: 控制去底的严格程度
- **调优建议**:
  - 背景色不均匀/有渐变 → 增大到 60-80
  - 背景色单一/对比度高 → 减小到 20-30
  - 物体边缘有锯齿 → 适当增大

### bg_threshold (背景二值化阈值)
- **默认值**: 240
- **作用**: 区分背景和前景的亮度界限
- **调优建议**:
  - 背景偏灰色 → 减小到 200-220
  - 背景纯白色 → 保持 240-250
  - 背景偏暗色 → 减小到 150-180

## 常见问题解决

### 问题1: 切出来的图片太多碎片
**原因**: min_area 设置过小，包含了噪点
**解决**: 增大 min_area 参数到 800-1500

### 问题2: 有些物体没有被检测到
**原因**: 物体与背景对比度不够或被过滤掉
**解决**: 
- 降低 bg_threshold 到 200-220
- 减小 min_area 到 300-400

### 问题3: 去底效果不好，边缘有白边
**原因**: tolerance 设置过小，没有完全去除背景
**解决**: 增大 tolerance 到 60-80

### 问题4: 物体内部被误删除
**原因**: tolerance 设置过大，误删了物体像素
**解决**: 减小 tolerance 到 20-30

## 最佳实践

### 输入图像要求
1. **背景色**: 尽量使用纯色背景（白色最佳）
2. **对比度**: 确保物体与背景有明显区别
3. **分辨率**: 建议 1000x1000 以上获得更好效果
4. **格式**: 支持 PNG、JPG、BMP 等常见格式

### 工作流程建议
1. **预处理**: 如果背景复杂，先用图像编辑软件简化背景
2. **参数测试**: 先用默认参数测试，再根据结果调优
3. **批量处理**: 相似图像可以使用相同参数批量处理
4. **后期优化**: 必要时手动调整个别切片的透明度

### 输出文件命名
- 格式: `sprite_000.png`, `sprite_001.png`, ...
- 使用3位数编号，支持最多999个物体
- 自动保存为PNG格式，保持透明度信息
