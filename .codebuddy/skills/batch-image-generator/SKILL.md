# Batch Image Generator

## Overview

Batch generate game assets from a structured JSON manifest file. This skill orchestrates multiple image generation skills (`text-to-image`, `generate-image-by-image`, `background-remover`) to produce complete asset sets in one automated run.

## When to Use This Skill

Use this skill when:
- Generating multiple game assets at once (characters, UI, backgrounds, effects)
- Need consistent style across all assets
- Want automated variant generation (hover/pressed states, animation frames)
- Require organized output with proper directory structure

## Quick Start

### Basic Usage

```bash
python3 scripts/batch_generate.py manifest.json
```

### With Options

```bash
# Dry run (show what would be generated)
python3 scripts/batch_generate.py manifest.json --dry-run

# Skip background removal for all assets
python3 scripts/batch_generate.py manifest.json --skip-remove-bg

# Custom output directory
python3 scripts/batch_generate.py manifest.json --output-dir ./assets

# Resume from a specific asset (skip already generated)
python3 scripts/batch_generate.py manifest.json --resume-from "animal_fox_01"

# Verbose output
python3 scripts/batch_generate.py manifest.json --verbose
```

## Manifest File Format

The manifest is a JSON file defining all assets to generate:

```json
{
  "config": {
    "style": "Q版卡通风格，可爱萌系，大头小身比例2:1，...",
    "default_size": "1024x1024",
    "default_quality": "high",
    "temp_dir": "./temp",
    "skills_dir": ".codebuddy/skills"
  },
  "assets": [
    {
      "id": "animal_panda_01",
      "type": "mother",
      "prompt": "可爱的Q版小熊猫，正面视角，...",
      "output": "images/characters/animal_panda_01.png",
      "size": "1024x1024",
      "quality": "high",
      "remove_bg": true
    },
    {
      "id": "animal_panda_02",
      "type": "variant",
      "mother_id": "animal_panda_01",
      "edit_prompt": "same pose, eyes closed, blinking, peaceful expression",
      "output": "images/characters/animal_panda_02.png",
      "strength": 0.5,
      "remove_bg": true
    },
    {
      "id": "bg_sky",
      "type": "mother",
      "prompt": "梦幻天空背景，淡蓝到白色渐变，...",
      "output": "images/environments/bg_sky.png",
      "size": "1408x640",
      "remove_bg": false
    }
  ]
}
```

### Config Section

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `style` | string | Yes | Global style prompt prepended to all generation prompts |
| `default_size` | string | No | Default image size (default: "1024x1024") |
| `default_quality` | string | No | Default quality level: low/medium/high (default: "high") |
| `temp_dir` | string | No | Temporary directory for intermediate files (default: "./temp") |
| `skills_dir` | string | No | Path to skills directory (default: ".codebuddy/skills") |

### Asset Types

#### Mother Image (`type: "mother"`)

Base image generated from text prompt.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Unique identifier for this asset |
| `type` | string | Yes | Must be `"mother"` |
| `prompt` | string | Yes | Text description (style will be prepended) |
| `output` | string | Yes | Output file path relative to output_dir |
| `size` | string | No | Image size (uses config.default_size if not set) |
| `quality` | string | No | Quality level (uses config.default_quality if not set) |
| `remove_bg` | boolean | No | Whether to remove background (default: true) |
| `background` | string | No | "transparent" or "opaque" (for text-to-image) |

#### Variant Image (`type: "variant"`)

Image generated by editing a mother image.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Unique identifier for this asset |
| `type` | string | Yes | Must be `"variant"` |
| `mother_id` | string | Yes | ID of the mother image to base this on |
| `edit_prompt` | string | Yes | Edit instruction for the variant |
| `output` | string | Yes | Output file path relative to output_dir |
| `strength` | float | No | Edit strength 0.0-1.0 (default: 0.5) |
| `remove_bg` | boolean | No | Whether to remove background (default: true) |

## CLI Parameters

| Parameter | Short | Description |
|-----------|-------|-------------|
| `manifest` | | Path to JSON manifest file (required) |
| `--output-dir` | `-o` | Base output directory (default: current dir) |
| `--dry-run` | `-d` | Show what would be generated without executing |
| `--skip-remove-bg` | | Skip background removal for all assets |
| `--resume-from` | `-r` | Resume from specified asset ID (skip previous) |
| `--verbose` | `-v` | Show detailed progress and debug info |
| `--parallel` | `-p` | Number of parallel workers (default: 1, sequential) |

## Output

The script produces:

1. **Generated assets** in the specified output paths
2. **Temp files** in the temp directory (can be cleaned up)
3. **Progress log** showing completion status
4. **Summary report** at the end with success/failure counts

## Error Handling

- If a generation fails, the script logs the error and continues with the next asset
- Use `--resume-from` to restart from a failed asset after fixing issues
- Failed assets are listed in the summary report
- Temp files are preserved for debugging (clean manually if needed)

## Best Practices

### Manifest Organization

1. **Group by category**: Put related assets together (all characters, then UI, then backgrounds)
2. **Mother before variants**: Always define mother images before their variants
3. **Consistent naming**: Use clear naming convention (e.g., `category_name_state.png`)

### Performance Tips

1. **Sequential execution**: API doesn't support parallel calls, so assets are generated one by one
2. **Resume support**: Use `--resume-from` to continue interrupted batches
3. **Dry run first**: Use `--dry-run` to validate manifest before actual generation

### Style Consistency

1. **Global style in config**: Define common style elements in `config.style`
2. **Asset-specific details**: Put unique details in each asset's `prompt`
3. **Variant consistency**: Use similar `strength` values for related variants
